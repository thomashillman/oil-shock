## Endpoint Failure Diagnosis

### Observed Failures
Three endpoints return HTTP 503 with non-JSON body "DNS cache overflow":

```
GET /health → HTTP 503, "DNS cache overflow"
GET /api/admin/rollout-status → HTTP 503, "DNS cache overflow"
GET /api/admin/rollout-readiness → HTTP 503, "DNS cache overflow"
GET /api/admin/api-health → HTTP 200 ✅ (ONLY WORKING ENDPOINT)
```

- `/api/admin/api-health` (worker/src/routes/admin-api-health.ts, line 179):
  - Returns JSON via `json()` helper

  - **IDENTICAL implementation pattern to failing endpoints**
  - **Works correctly (HTTP 200)**

### Root Cause Analysis

**This is a Cloudflare platform-level issue, NOT a code problem.**

Evidence:
1. **Plain text error body** ("DNS cache overflow") indicates HTTP response is generated by Cloudflare edge, not worker code
2. **HTTP 503 before JSON** suggests platform rejection before route code executes
3. **Identical code patterns** between working (`/api/admin/api-health`) and failing endpoints (`/api/admin/rollout-status`, `/api/admin/rollout-readiness`)
4. **/health endpoint failure** (no auth check, simple code) proves this is not route-specific
5. **Mixed success/failure** on similar endpoints suggests DNS resolution or Cloudflare deployment configuration issue

### Likely Causes (Requires Infrastructure Investigation)

1. **Cloudflare DNS cache misconfiguration** — Some routes cached or broken in edge network
2. **Preview worker deployment state** — Partial deployment or stale routes
3. **DNS resolver overload** — "DNS cache overflow" could be from Cloudflare's resolver hitting limits
4. **Regional edge cache issue** — Specific edge locations returning stale/broken responses

### Next Steps (Infrastructure, Not Code)

This requires Cloudflare investigation:
- Check Cloudflare dashboard for deployment status of `energy-dislocation-engine-preview` worker
- Verify all routes are deployed and active
- Check DNS resolution for `energy-dislocation-engine-preview.tj-hillman.workers.dev`
- Clear Cloudflare DNS/routing cache if available
- Monitor edge error rates and deployment logs
- Consider redeploying worker to clear any stale state
